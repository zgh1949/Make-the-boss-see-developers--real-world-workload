优化后完整需求文档
一、需求定位与目标（核心优化补充）
核心目标
面向产品经理、测试人员、非技术管理层，通过可视化单步演示，直观讲清「前端页面操作 → JSON 数据传输 → 后端 API 处理 → 数据库读写」的完整 Web 请求链路，降低技术认知门槛，建立全流程共识。
受众分层适配
管理层 / 非技术岗：看懂数据流转的完整路径，理解 “页面输入的内容最终如何存储、如何查询”
产品岗：理解接口、参数、数据结构与页面功能的对应关系，建立 “功能→数据” 的认知
测试岗：直观看到接口入参、SQL 执行、数据落库的全流程逻辑，辅助理解测试边界
使用场景
内部技术科普、需求评审开场通识、新人入职技术入门讲解
二、整体布局优化
保留三栏核心结构，补充全局导航与控制区，形成「顶部总览 + 三栏主体 + 底部控制」的三层结构，解决原需求 “三栏割裂、无全局视角” 的问题：
顶部：全链路进度条
固定展示 5 个流程节点：前端页面 → 网络传输 → 后端处理 → 数据库交互 → 返回前端
演示过程中，当前执行到的节点同步高亮，让观众随时建立全局认知，清晰知道 “现在数据走到了哪一步”
主体：三等分三栏布局
左栏：前端页面区（记事本应用）
中栏：后端代码区（Python 伪代码 + 单步调试）
右栏：数据库区（MySQL 表模拟 + 数据变化动效）
底部：演示控制栏
集成场景选择、操作按钮、步骤说明，统一控制入口，避免操作分散
三、分模块需求细化
（一）左栏：前端页面区（记事本应用）
基础能力保留：笔记列表展示、按标题模糊查询、新增笔记保存
优化补充点：
操作与演示双向联动
支持两种触发方式：① 直接在页面操作（点击保存 / 查询 / 单条笔记），自动触发对应接口的全链路演示；② 底部选择场景触发演示
笔记列表支持点击单条条目，触发「get 详情接口」的完整演示
JSON 报文显性化展示
页面内置可折叠的「请求 / 响应报文区」
演示过程中实时展示当前接口的请求 JSON和响应 JSON，关键字段高亮标注
核心价值：直观展示 “页面表单里的文字，最终变成了结构化的 JSON 数据发给后端”，帮非技术人员理解 “接口传的是什么”
统计结果展示位
固定展示笔记总数，调用 statistic 接口时同步高亮刷新，对应统计接口的返回结果
（二）中栏：后端代码区（Python 伪代码）
基础要求保留：类 Python 伪代码、无需真实运行、单步调试游标效果、4 个接口（save/get/list/statistic）、总代码控制在 80 行以内
优化补充点：
代码规范与可读性优化
采用类 Flask 路由风格，接口结构清晰，每行代码简洁易懂，剔除冗余技术细节
支持基础语法着色（关键字、函数名、字符串、注释区分颜色），降低阅读门槛
四个接口职责明确，完整覆盖增、查、列表、统计四类典型操作：
save：接收 JSON 参数，执行插入 SQL，返回成功结果
get：接收 ID 参数，执行主键查询 SQL，返回单条数据
list：接收关键词参数，执行模糊查询 SQL，返回列表数据
statistic：无业务参数，执行 count 聚合 SQL，返回统计结果
单步调试能力细化
代码左侧带行号，当前执行行背景高亮，模拟调试游标效果
单步粒度为 “一行代码一步”，点击 “下一步” 逐行执行，支持逐行讲解每行代码的作用
运行时变量面板
代码区下方固定展示 3 个核心变量，随执行步骤实时更新：
请求入参：展示前端传来的参数内容
执行 SQL：展示后端拼接后、即将发给数据库的 SQL 语句
返回结果：展示接口最终返回的数据
核心价值：把代码背后 “看不见的中间数据” 显性化，避免观众只看代码、不懂逻辑
接口快速切换
切换演示场景时，代码区自动切换对应接口的伪代码，无需手动修改
（三）右栏：数据库区（MySQL 模拟）
基础要求保留：表格样式模拟数据表、插入过程高亮动画
优化补充点：
表结构前置展示
数据表格上方展示表结构定义（字段名、类型、注释），先让观众看懂 “数据存在什么样的结构里”
分场景差异化动效
不同操作对应不同的高亮动效，直观区分读写类型，观众一眼就能分辨当前是读还是写：
新增操作（save 接口）：新行插入后整行绿色渐变高亮 1.5 秒后恢复，模拟数据落库过程
查询操作（get/list 接口）：命中的行蓝色闪烁 2 次，模拟数据被检索命中
统计操作（statistic 接口）：表格下方的统计结果区黄色高亮，展示 count 计算结果
三端数据一致性
数据库表中的数据，与前端列表、后端代码逻辑严格保持同步
新增一条数据，三端同步更新；查询过滤时，前端列表与数据库命中行同步联动
四、交互演示流程优化（核心升级项）
双演示模式
单步模式：默认模式，点击「下一步」执行一个步骤，适合讲解时逐行拆解、随时停顿答疑
自动播放模式：点击后按固定间隔自动执行全流程，适合快速展示、无人讲解场景
四大标准演示场景
每个接口对应一个独立场景，可通过底部下拉框一键切换，覆盖最典型的四类后端操作：
场景 1：新增笔记（save 接口）—— 完整演示 “数据写入” 全链路
场景 2：查看详情（get 接口）—— 完整演示 “单条数据读取” 全链路
场景 3：搜索列表（list 接口）—— 完整演示 “条件过滤查询” 全链路
场景 4：数量统计（statistic 接口）—— 完整演示 “聚合计算” 全链路
全链路同步机制
每执行一步，三栏 + 顶部进度条同步联动，避免各模块各自动画、观众认知混乱：
顶部进度条跳到对应节点
后端代码高亮当前执行行
变量面板更新当前中间数据
数据库区执行对应动效（如有）
前端区同步更新页面状态（如有）
底部显示当前步骤的大白话说明
重置与复用能力
支持一键重置：数据恢复初始状态、代码游标归零、进度条重置、演示回到初始状态
支持重复演示同一场景，无需刷新页面
五、科普向体验优化（面向非技术受众的专项优化）
步骤大白话提示
底部控制栏同步显示当前步骤的通俗解释，讲解人可直接参考，避免纯技术术语堆砌。
示例：执行到数据库插入时，提示 “后端把拼接好的 SQL 语句发给数据库，正式把数据写入表中”
术语悬浮解释
对 “API”“JSON”“SQL” 等技术术语，鼠标悬浮时显示通俗解释，降低非技术受众的认知门槛。
数据流向引导
关键节点切换时，栏与栏之间出现短暂的箭头动效，直观体现 “数据从左向右流转” 的方向。
六、技术实现约束（保留并明确）
产出为单个 HTML 文件，所有 CSS、JS 代码内嵌其中，无需任何外部依赖
纯原生 HTML+CSS+JavaScript 实现，双击即可打开运行，兼容 Chrome、Edge 等主流浏览器
无需真实后端服务、无需真实数据库，所有逻辑均在前端模拟实现
整体适配 1920*1080 分辨率屏幕，演示时无横向滚动条
