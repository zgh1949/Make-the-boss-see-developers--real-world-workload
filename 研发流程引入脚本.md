# 产品研发流程引入脚本（画面逐步展示版）

## 一、整体思路

- **不是游戏、不是对话**，而是一段「视觉叙事」：同一个备忘录应用，在画面里一步步从构想变成可运行的产品。
- **核心画布**：画面中央始终是一个「备忘录应用」的示意，随幕次推进而变形——
  `构想示意 → 黑白线框原型 → 彩色设计稿 → 前端代码+实时预览 → 前后端+数据库全貌`。
- **推进方式**：用户点击「下一步」（或自动播放），每一步画面发生一处可见变化，底部一行简短字幕说明。
- **角色立绘**：4 张 PNG（`boss.png` / `pm.png` / `ui.png` / `dev.png`），当前幕的角色立绘显示在画面左侧，其余隐藏。
- **结尾**：自然过渡到现有的 `index.html`「Web 请求全链路演示」。

> 备忘录布局（贯穿所有阶段，只是视觉处理不同）：
> ① 顶部标题栏「备忘录」+ 笔记总数　② 搜索框　③ 笔记列表（3~4 行：标题/摘要/时间）　④「新建」按钮　⑤ 笔记详情区

## 二、分幕脚本

### 第一幕 · 老板的构想

**角色**：老板（boss.png）
**画面起点**：空白画布。

#### 步骤 1-1
- **字幕**：老板想做一个像苹果备忘录一样的软件。
- **画面变化**：老板立绘从左侧淡入；画面中央淡入一张「苹果备忘录」参考图（干净的笔记列表截图/示意）。

#### 步骤 1-2
- **字幕**：他把这个想法交给了产品经理。
- **画面变化**：老板立绘旁出现一个「需求」气泡/箭头，指向右下；产品经理立绘淡入接住。参考图保留在中央。

---

### 第二幕 · 产品经理做原型

**角色**：产品经理（pm.png）
**画面起点**：中央的参考图淡出，换成一张「白纸/线框画板」。

> 说明：本幕用灰底 + 虚线白块表示线框，逐块淡入，模拟"画原型"的过程。

#### 步骤 2-1
- **字幕**：产品经理先搭出页面结构——标题栏。
- **画面变化**：画板顶部出现标题栏线框（虚线矩形 + 占位文字「备忘录」）。

#### 步骤 2-2
- **字幕**：加上搜索框。
- **画面变化**：标题栏下方出现搜索框线框（虚线矩形 + 🔍 占位）。

#### 步骤 2-3
- **字幕**：再画出笔记列表。
- **画面变化**：搜索框下方依次淡入 3 行笔记线框（每行：小方块 + 两根占位线条）。

#### 步骤 2-4
- **字幕**：还有「新建」按钮和总数。
- **画面变化**：右下角出现「+ 新建」线框按钮；标题栏右侧出现「总数: N」占位。

#### 步骤 2-5
- **字幕**：点开某条能看详情——原型完成，交给 UI。
- **画面变化**：其中一行笔记高亮，右侧淡出详情区线框（大标题 + 几根正文线条）；产品经理立绘把画板推向右侧，UI 立绘淡入。

---

### 第三幕 · UI 设计

**角色**：UI（ui.png）
**画面起点**：第二幕的线框原位保留。

> 说明：本幕线框「原地变身」——上色、圆角、阴影、字体，逐步变成设计稿。

#### 步骤 3-1
- **字幕**：UI 把原型变成好看的设计稿——先定主色。
- **画面变化**：标题栏、按钮填上蓝色主色调；虚线边框变为实线圆角。

#### 步骤 3-2
- **字幕**：列表做成卡片，加点阴影和留白。
- **画面变化**：3 行线框变成白色圆角卡片，加投影、间距拉大，占位线条变灰色文字。

#### 步骤 3-3
- **字幕**：搜索框、按钮也统一样式。
- **画面变化**：搜索框变圆角输入框带图标；「新建」按钮变实心蓝色胶囊按钮。

#### 步骤 3-4
- **字幕**：详情区同样美化——设计稿完成，交给程序员。
- **画面变化**：详情区线框变成带配色排版的设计稿；UI 立绘把设计稿推向右侧，程序员立绘淡入。

---

### 第四幕 · 前端程序员写代码

**角色**：程序员（dev.png）
**画面起点**：画面分为左右两栏——左栏「代码编辑器」，右栏保留第三幕的设计稿（作为目标）。

> 说明：左栏代码逐行出现，右栏的"预览"从空白逐步长成设计稿的样子，体现"代码把设计变成真实页面"。

#### 步骤 4-1
- **字幕**：前端程序员把设计稿变成真正的代码——先写结构。
- **画面变化**：左栏逐行淡入 HTML 结构代码（`<header>`、`<input>`、`<ul><li>`…）；右栏预览同步出现对应的标题栏、搜索框、列表骨架（无样式）。

#### 步骤 4-2
- **字幕**：再写样式，让页面变好看。
- **画面变化**：左栏继续淡入 CSS 代码（颜色、圆角、阴影…）；右栏预览从灰色骨架「上色」变成第三幕的设计稿样式。

#### 步骤 4-3
- **字幕**：最后加上交互——点击、搜索、新建。
- **画面变化**：左栏淡入 JS 代码（`onclick`、`fetch`…）；右栏预览的按钮/列表出现可交互提示（hover 高亮、点击波纹）。

---

### 第五幕 · 后端程序员写接口

**角色**：程序员（dev.png，同一立绘）
**画面起点**：第四幕画面右下扩展出一块「后端 + 数据库」区域。

> 说明：前端已就绪，后端同步写出四个接口，并出现数据库表。

#### 步骤 5-1
- **字幕**：后端同步写出对应的接口——保存笔记。
- **画面变化**：后端区出现 `save` 接口代码块；数据库区出现 notes 表，插入一行（高亮）。

#### 步骤 5-2
- **字幕**：查看详情。
- **画面变化**：出现 `get` 接口代码块；数据库某一行高亮（被查询命中）。

#### 步骤 5-3
- **字幕**：搜索列表。
- **画面变化**：出现 `list` 接口代码块；数据库多行高亮（模糊命中）。

#### 步骤 5-4
- **字幕**：数量统计。
- **画面变化**：出现 `statistic` 接口代码块；数据库底部出现 `COUNT(*) = N`。

#### 步骤 5-5
- **字幕**：前后端就绪，来看一个请求是怎么跑通的。
- **画面变化**：画面收拢为四格缩略图——「前端页面 / JSON 报文 / 后端接口 / 数据库」，中间出现流动箭头。

---

### 第六幕 · 过渡到全链路演示

#### 步骤 6-1
- **字幕**：页面 → JSON → 后端 → 数据库 → 返回前端，一条完整链路。
- **画面变化**：四格之间依次亮起流向箭头（左→右→左）。

#### 步骤 6-2
- **字幕**：（按钮）进入 Web 请求全链路演示
- **画面变化**：画面中央出现「进入演示」按钮 → 点击跳转 `index.html`。

## 三、画面状态总览（给实现用）

| 阶段 | 画布表现 | 角色 |
|---|---|---|
| 第一幕 | 苹果备忘录参考图 | 老板 |
| 第二幕 | 灰底虚线线框（逐块出现） | 产品经理 |
| 第三幕 | 线框上色变设计稿（圆角/阴影/配色） | UI |
| 第四幕 | 左代码 + 右预览（骨架→上色→可交互） | 程序员 |
| 第五幕 | 前端 + 后端代码 + 数据库表 | 程序员 |
| 第六幕 | 四格缩略 + 流向箭头 → 演示入口 | — |

## 四、实现备注

- 建议单 HTML 实现，固定画布区域，每一步用 JS 切换画布的「状态」（显示/隐藏/样式类），CSS 过渡实现变形动画。
- 字幕区固定在画布下方，一行；角色立绘固定在画布左侧。
- 步骤总数约 20 步，可「下一步」手动推进，也可加「自动播放」。
- 备忘录参考图、线框、设计稿、代码均可纯 HTML/CSS 绘制（无需真实图片），保证单文件可运行。
- 末尾按钮：`onclick="location.href='index.html'"`。
